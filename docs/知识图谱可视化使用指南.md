# 知识图谱可视化探索系统 - 使用指南

## 🎯 功能概述

本系统提供类似 Neo4j Browser 的交互式知识图谱可视化功能，用户无需编写 Cypher 查询语句，通过图形界面即可探索中医知识图谱。

---

## ✨ 核心功能

### 1. 🔍 **实体搜索**
- 在顶部搜索框输入实体名称（如"感冒"）
- 按回车或点击搜索按钮
- 自动展示该实体及其关联的知识网络

### 2. 📊 **筛选探索**
左侧筛选面板提供：
- **实体类型筛选**：勾选感兴趣的实体类型（Disease, Drug, Food 等）
- **关系类型筛选**：选择特定的关系类型（common_drug, has_symptom 等）
- **数量限制**：通过滑块控制显示的节点数量（10-500）
- 点击"应用"按钮执行筛选查询

### 3. 🔗 **节点展开**
- **单击任意节点**，自动展开其一度关联的节点
- 支持多次点击，逐层深入探索
- 新展开的节点会自动合并到现有图谱中

### 4. 📈 **统计信息**
右上角实时显示：
- 当前视图中的节点数量
- 当前视图中的关系数量
- 可随时隐藏/显示统计卡片

### 5. 🕒 **搜索历史**
- 自动记录最近 10 次搜索
- 点击历史标签快速重新搜索
- 支持删除单个历史或清空全部历史
- 历史记录本地存储，刷新页面不丢失

### 6. 💾 **数据导出**
- 点击"导出数据"按钮
- 导出当前视图的所有节点和关系
- JSON 格式，便于后续分析和分享

### 7. 🎨 **交互操作**
- **拖拽节点**：调整节点位置
- **滚轮缩放**：放大/缩小视图
- **双击画布**：重置视角
- **悬停提示**：查看节点和关系的详细信息

## 🚀 快速开始

### 方式一：搜索特定实体
1. 打开知识图谱可视化页面
2. 在搜索框输入"感冒"
3. 点击搜索或按回车
4. 查看感冒相关的药品、症状、疗法等知识网络

### 方式二：筛选探索
1. 在左侧面板勾选"Disease"和"Drug"
2. 勾选"common_drug"关系类型
3. 调整数量限制为 50
4. 点击"应用"查看疾病与药品的关系图

### 方式三：逐层深入
1. 搜索"感冒"
2. 点击"阿司匹林"节点，展开其更多关联
3. 继续点击其他感兴趣的节点
4. 构建个性化的知识探索路径

## 📖 使用场景

### 场景一：学习中医知识
```
目标：了解感冒的完整治疗方案
步骤：
1. 搜索"感冒"
2. 查看常用药品（common_drug）
3. 查看推荐食物（recommand_eat）
4. 查看治疗方法（cure_way）
5. 查看相关症状（has_symptom）
```

### 场景二：研究药品关联
```
目标：研究某药品的适用范围
步骤：
1. 搜索药品名称
2. 点击节点展开关联
3. 查看治疗哪些疾病
4. 查看是否有禁忌食物
```

### 场景三：数据分析
```
目标：分析特定类型的实体分布
步骤：
1. 只勾选感兴趣的实体类型
2. 只勾选特定关系类型
3. 调整显示数量
4. 导出数据进行进一步分析
```

## 🎯 高级技巧

### 1. **组合筛选**
- 同时选择多个实体类型和关系类型
- 例如：只看 Disease→Drug→Food 的治疗链条

### 2. **分步探索**
- 先搜索核心实体
- 逐步点击展开关联节点
- 避免一次性加载过多数据

### 3. **性能优化**
- 初次浏览建议限制数量在 50 以内
- 需要详细分析时再增加数量
- 使用"重置视图"清理画布重新开始

### 4. **数据导出后的处理**
```javascript
// 导出的 JSON 格式
{
  "nodes": [
    {"id": "感冒", "name": "感冒", "category": "Disease"},
    {"id": "阿司匹林", "name": "阿司匹林", "category": "Drug"}
  ],
  "links": [
    {"source": "感冒", "target": "阿司匹林", "name": "common_drug"}
  ],
  "timestamp": "2026-04-03T..."
}
```

## 📝 常见问题

### Q1: 为什么点击节点没有反应？
A: 请确保：
1. 后端服务正在运行
2. Neo4j 数据库已连接
3. 该节点确实有关联关系

### Q2: 筛选后没有数据显示？
A: 检查：
1. 是否选择了正确的实体类型
2. 是否选择了正确的关系类型
3. 尝试放宽筛选条件或增加数量限制

### Q3: 图谱太大导致浏览器卡顿？
A: 建议：
1. 减少显示数量限制（如 50 个）
2. 使用分步探索，不要一次性展开太多
3. 点击"重置视图"重新开始

### Q4: 如何保存当前的图谱状态？
A: 点击"导出数据"按钮，保存为 JSON 文件

### Q5: 搜索历史在哪里查看？
A: 左侧面板底部，显示最近的搜索记录
