# 数据管理模块开发总结

## 概述
在知识图谱可视化功能基础上，实现了实体和关系的完整CRUD（增删改查）功能。

---

## 后端实现

### 1. Neo4j客户端增强
**文件**: `backend/app/services/kg/neo4j_client.py`

#### 新增方法

| 方法 | 功能 | 参数 |
|------|------|------|
| `delete_entity()` | 删除实体及其关系 | entity_name |
| `update_entity()` | 更新实体属性 | entity_name, category, properties |
| `list_entities()` | 查询实体列表（分页） | category, limit, offset |
| `list_relations()` | 查询关系列表 | source_entity, target_entity, relation_type, limit |
| `delete_relation()` | 删除关系 | source_entity, target_entity, relation_type |
| `update_relation()` | 更新关系 | source, target, old_type, new_type, properties |
| `get_entity_types()` | 获取所有实体类型 | - |
| `get_relation_types()` | 获取所有关系类型 | - |

### 2. API接口完善
**文件**: `backend/app/api/kg_management.py`

#### 新增接口

**实体CRUD**
```
GET    /api/kg/entities              - 查询实体列表（支持分页、筛选）
GET    /api/kg/entities/{name}       - 查询实体关系（用于可视化）
POST   /api/kg/entities              - 创建实体
PUT    /api/kg/entities/{name}       - 更新实体
DELETE /api/kg/entities/{name}       - 删除实体
```

**关系CRUD**
```
GET    /api/kg/relations                         - 查询关系列表
POST   /api/kg/relations                         - 创建关系
PUT    /api/kg/relations/{source}/{target}/{type} - 更新关系
DELETE /api/kg/relations/{source}/{target}/{type} - 删除关系
```

**辅助接口**
```
GET /api/kg/entity-types    - 获取所有实体类型
GET /api/kg/relation-types  - 获取所有关系类型
GET /api/kg/stats           - 获取统计信息
```

## 数据模型

### 实体模型
```python
class EntityCreate(BaseModel):
    name: str                # 实体名称
    category: str            # 实体类型（如Disease, Symptom等）
    description: Optional[str] = None
    properties: Optional[dict] = {}
```

### 关系模型
```python
class RelationCreate(BaseModel):
    source_entity: str       # 源实体名称
    target_entity: str       # 目标实体名称
    relation_type: str       # 关系类型（如HAS_SYMPTOM, TREATS等）
    properties: Optional[dict] = {}
```

## 使用示例

### 1. 新增实体
```bash
POST http://localhost:8000/api/kg/entities
Content-Type: application/json

{
  "name": "新疾病",
  "category": "Disease",
  "description": "这是一个新疾病",
  "properties": {}
}
```

### 2. 查询实体列表
```bash
GET http://localhost:8000/api/kg/entities?category=Disease&limit=20&offset=0
```

### 3. 新增关系
```bash
POST http://localhost:8000/api/kg/relations
Content-Type: application/json

{
  "source_entity": "慢性胃炎",
  "target_entity": "胃痛",
  "relation_type": "HAS_SYMPTOM",
  "properties": {}
}
```

### 4. 删除实体
```bash
DELETE http://localhost:8000/api/kg/entities/疾病名称
```

## 技术要点

### 1. Neo4j查询优化
- 使用 `DETACH DELETE` 级联删除关系
- 分页查询使用 `SKIP/LIMIT`
- 关系查询使用模式匹配优化

### 2. 前端分页策略
- 实体：后端分页（数据库层面）
- 关系：前端分页（内存过滤）

### 3. 错误处理
- 统一使用HTTP状态码
- 404：实体/关系不存在
- 500：服务器错误
- 前端统一错误提示

### 4. 安全防护
- 删除操作需要二次确认
- URL参数编码（encodeURIComponent）
- 表单输入验证
