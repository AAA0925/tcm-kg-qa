## 📋 概述

本文档详细说明如何收集、整理中医知识图谱所需的数据。

## 🎯 数据来源

### 1. 核心数据源 - 《中医临床诊疗术语》

**重要性**: ⭐⭐⭐⭐⭐  
**数据类型**: 半结构化文本  
**包含内容**:
- 疾病术语（含病因、病类、症状）
- 证候术语（含症状、治法）
- 治法术语（含功效、适应证候）

**获取方式**:
1. 下载 PDF/Word 版本
2. 使用 OCR 或手动整理为文本格式
3. 按照指定格式保存

**文件格式要求**:

```
# disease_terms.txt - 疾病术语
疾病名称 | 病因 | 病类 | 症状
痛风 | 湿热蕴结 | 筋骨病类 | 关节红肿热痛，发热
水痘 | 外感时邪 | 皮肤病类 | 皮疹，瘙痒

# syndrome_terms.txt - 证候术语
证候名称 | 症状 | 治法
津伤证 | 口干，咽燥 | 滋阴生津
风热血燥证 | 皮疹色红，瘙痒 | 疏风清热

# therapy_terms.txt - 治法术语
治法名称 | 功效 | 适应证候
清暑宣肺 | 清解暑热 | 暑湿犯肺证
行气和胃 | 调理气机 | 脾胃不和证
```

---

### 2. 网页数据 - 药智网、中医中药网、A+ 医学百科

**重要性**: ⭐⭐⭐⭐  
**数据类型**: 半结构化网页  
**包含内容**:
- 方剂数据（方名、出处、处方、炮制、功用、主治）
- 药物数据（名称、别名、性味、归经、功效、主治）

**爬取策略**:

#### 药智网方剂数据
```python
# 运行爬虫
cd backend
python data_collector.py

# 输出: data/raw/web_prescriptions.json
```

#### 多网站药物数据
```python
# 自动爬取三个网站
python -c "from data_collector import WebCrawlerCollector; c = WebCrawlerCollector(); c.crawl_herbs_from_sites()"

# 输出: data/raw/web_herbs.json
```

**注意事项**:
- ⚠️ 遵守 robots.txt
- ⚠️ 控制爬取频率（建议 1 秒/页）
- ⚠️ 仅用于学术研究

## 🔧 数据处理流程

### Step 1: 数据收集

```bash
cd backend

# 1. 准备《中医临床诊疗术语》文件
# 将文件复制到 data/raw/ 目录
cp /path/to/terms.txt data/raw/disease_terms.txt

# 2. 运行数据收集器
python data_collector.py

# 输出目录: data/raw/
# - clinical_diseases.json
# - clinical_syndromes.json
# - clinical_therapies.json
# - web_prescriptions.json
# - web_herbs.json
```

### Step 2: 准备实体别名字典

创建 `data/raw/aliases.txt`:

```
# 格式：标准名 | 别名 1, 别名 2, 别名 3
阴虚血热证 | 阴虚血热，血虚阳浮发热证
肝阳上亢证 | 肝阳上亢，肝阳偏亢
小柴胡汤 | 柴胡汤
黄连 | 川连，味连
```

### Step 3: 数据清洗和处理

```bash
# 运行数据处理器
python data_processor.py

# 处理步骤:
# 1. 文本清洗（去除特殊字符、规范化标点）
# 2. 实体对齐（使用字典法统一名称）
# 3. 数据去重
# 4. 提取药物组成
# 5. 合并数据源

# 输出目录：data/processed/
# - processed_diseases.json
# - processed_prescriptions.json
# - processed_herbs.json
# - merged_tcm_data.json
```

### Step 4: 数据验证

```python
# 检查数据质量
import json

with open('data/processed/merged_tcm_data.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

print(f"疾病：{len(data['diseases'])} 条")
print(f"方剂：{len(data['prescriptions'])} 条")
print(f"药物：{len(data['herbs'])} 条")

# 抽样检查
import random
sample = random.choice(data['prescriptions'])
print(f"\n示例方剂：{sample['name']}")
print(f"组成：{sample['composition']}")
print(f"功效：{sample['effects']}")
```

## 📊 数据结构定义

### 疾病实体 (Disease)
```json
{
  "name": "疾病名称",
  "category": "Disease",
  "cause": "病因",
  "disease_class": "病类",
  "symptoms": ["症状 1", "症状 2"]
}
```

### 证候实体 (Syndrome)
```json
{
  "name": "证候名称",
  "category": "Syndrome",
  "symptoms": ["症状 1", "症状 2"],
  "therapy": "治法"
}
```

### 治法实体 (Therapy)
```json
{
  "name": "治法名称",
  "category": "Therapy",
  "effect": "功效",
  "syndromes": ["证候 1", "证候 2"]
}
```

### 方剂实体 (Prescription)
```json
{
  "name": "方剂名称",
  "category": "Prescription",
  "alias": "别名",
  "composition": ["药物 1", "药物 2"],
  "source": "出自典籍",
  "effects": "功效",
  "indications": "主治"
}
```

### 药物实体 (Herb)
```json
{
  "name": "药物名称",
  "category": "Herb",
  "alias": "别名",
  "property": "性味",
  "meridian": "归经",
  "effects": "功效",
  "indications": "主治"
}
```

## 🎯 质量要求

### 数据准确性
- ✅ 来源权威（《中医临床诊疗术语》为核心）
- ✅ 人工校对（关键数据必须人工审核）
- ✅ 多方验证（不同来源交叉验证）

### 数据完整性
- ✅ 必填字段不能为空
- ✅ 列表字段至少一项
- ✅ 重要信息不遗漏

### 数据一致性
- ✅ 名称统一（使用别名字典对齐）
- ✅ 格式规范（统一的 JSON 格式）
- ✅ 编码一致（UTF-8）

## ⚠️ 常见问题

### Q1: 数据从哪里开始收集？
**A**: 优先收集《中医临床诊疗术语》，这是核心数据源。然后补充方剂和药物数据。

### Q2: 如何处理同名异方的情况？
**A**: 通过"别名"字段区分，并在构建图谱时添加更多属性（如组成、功效）来唯一标识。

### Q3: 实体对齐的字典怎么建立？
**A**: 
1. 从专业书籍中整理
2. 参考中医辞海等工具书
3. 在数据收集中逐步积累

### Q4: 爬虫被网站封禁怎么办？
**A**:
1. 降低爬取频率
2. 使用多个 IP
3. 考虑购买商业数据
4. 寻求实验室合作

### Q5: 数据量应该达到多少？
**A**: 根据硕士论文要求，建议：
- 疾病：500+
- 证候：300+
- 治法：200+
- 方剂：2000+
- 药物：1000+
- 总实体数：4000+
- 总关系数：25000+
