# 中医知识图谱 Schema 设计文档

## 📋 概述

本文档定义了基于 `health-sample.owl` 数据源的中医知识图谱的实体类型、关系类型和属性设计。

**数据来源**: health-sample.owl (中医养生本体)  
**数据规模**: 5,155 个实例，105 种关系，13,613 个三元组  
**设计原则**: 以中医养生为核心，兼顾临床诊疗，适合本科毕设需求

---

## 🎯 实体类型设计 (Entity Types)

根据数据分析，定义 **10 类核心实体**：

### 1. Disease (疾病)
- **描述**: 中医或现代医学的疾病名称
- **示例**: 咳嗽、胃痛、感冒、失眠
- **预期数量**: ~500-800
- **属性**:
  - `name`: 疾病名称
  - `description`: 疾病描述
  - `category`: 疾病分类（如：呼吸系统疾病）

### 2. Syndrome (证候)
- **描述**: 中医辨证论治中的证候类型
- **示例**: 气虚证、脾胃虚寒、肝阳上亢
- **预期数量**: ~200-300
- **属性**:
  - `name`: 证候名称
  - `description`: 证候描述

### 3. Prescription (方剂)
- **描述**: 中药方剂，包括汤剂、丸剂、散剂等
- **示例**: 香砂六君子汤、如意定喘丸、通宣理肺丸
- **预期数量**: ~300-500
- **属性**:
  - `name`: 方剂名称
  - `alias`: 别名
  - `category`: 方剂分类（如：补气剂、行气剂）
  - `source`: 出处典籍

### 4. Herb (药物)
- **描述**: 中药材
- **示例**: 人参、当归、柴胡、甘草
- **预期数量**: ~200-400
- **属性**:
  - `name`: 药物名称
  - `alias`: 别名
  - `property`: 性味（寒、热、温、凉）
  - `meridian`: 归经

### 5. Symptom (症状)
- **描述**: 疾病或证候表现出的症状
- **示例**: 食欲不振、面色萎黄、胸闷、气短
- **预期数量**: ~500-800
- **属性**:
  - `name`: 症状名称
  - `description`: 症状描述

### 6. Therapy (治法/疗法)
- **描述**: 中医治疗方法，包括内治法和外治法
- **示例**: 消法、艾灸、针灸、推拿
- **预期数量**: ~100-200
- **属性**:
  - `name`: 治法名称
  - `type`: 治法类型（内治/外治）
  - `description`: 治法描述

### 7. WellnessMethod (养生方法)
- **描述**: 中医养生保健方法
- **示例**: 饮食养生、运动养生、情志调养、气功健身
- **预期数量**: ~300-500
- **属性**:
  - `name`: 养生方法名称
  - `category`: 养生类别（饮食/运动/情志等）
  - `operation`: 操作方法
  - `frequency`: 使用频次

### 8. Meridian (经络)
- **描述**: 中医经络系统
- **示例**: 心经、脾经、肺经、三焦
- **预期数量**: ~20-50
- **属性**:
  - `name`: 经络名称
  - `type`: 经络类型（经脉/络脉）

### 9. Property (性味)
- **描述**: 药物的性味特征
- **示例**: 寒、热、温、凉、平、甘、苦、辛
- **预期数量**: ~20-30
- **属性**:
  - `name`: 性味名称
  - `category`: 类别（四气/五味）

### 10. Classic (典籍)
- **描述**: 中医经典著作
- **示例**: 《伤寒论》、《本草纲目》、《孙真人食忌》
- **预期数量**: ~50-100
- **属性**:
  - `name`: 典籍名称
  - `author`: 作者
  - `dynasty`: 朝代

## 🔗 关系类型设计 (Relation Types)

基于 OWL 本体中的 105 种关系，筛选出 **18 种核心关系**：

### 疾病相关关系

#### 1. has_symptom (有症状)
- **头实体**: Disease
- **尾实体**: Symptom
- **描述**: 疾病表现出的症状
- **示例**: 感冒 → 有症状 → 发热
- **预期数量**: ~1000-1500

#### 2. corresponds_to_syndrome (对应证候)
- **头实体**: Disease
- **尾实体**: Syndrome
- **描述**: 疾病对应的中医证候
- **示例**: 胃痛 → 对应证候 → 脾胃虚寒证
- **预期数量**: ~200-300

#### 3. treated_by_therapy (采用治法)
- **头实体**: Disease / Syndrome
- **尾实体**: Therapy
- **描述**: 疾病或证候采用的治疗方法
- **示例**: 气虚证 → 采用治法 → 补气法
- **预期数量**: ~300-500

### 方剂相关关系

#### 4. treats (治疗)
- **头实体**: Prescription
- **尾实体**: Disease / Symptom / Syndrome
- **描述**: 方剂治疗的疾病、症状或证候
- **示例**: 香砂六君子汤 → 治疗 → 脾胃虚寒
- **预期数量**: ~500-800
- **数据来源**: OWL 中"治疗"关系（57次使用）

#### 5. composed_of (由...组成)
- **头实体**: Prescription
- **尾实体**: Herb
- **描述**: 方剂的药物组成
- **示例**: 香砂六君子汤 → 由...组成 → 人参
- **预期数量**: ~1000-1500

#### 6. belongs_to_category (属于分类)
- **头实体**: Prescription
- **尾实体**: Category (作为方剂的分类标签)
- **描述**: 方剂的分类（上位词关系）
- **示例**: 香砂六君子汤 → 属于分类 → 补气剂
- **预期数量**: ~300-500
- **数据来源**: OWL 中"上位词"关系（313次使用）

#### 7. from_classic (出自典籍)
- **头实体**: Prescription
- **尾实体**: Classic
- **描述**: 方剂的出处
- **示例**: 某方 → 出自典籍 → 《伤寒论》
- **预期数量**: ~100-200
- **数据来源**: OWL 中"来源"关系（157次使用）

### 药物相关关系

#### 8. has_property (具有性味)
- **头实体**: Herb
- **尾实体**: Property
- **描述**: 药物的性味特征
- **示例**: 黄连 → 具有性味 → 寒
- **预期数量**: ~200-400

#### 9. belongs_to_meridian (归经)
- **头实体**: Herb
- **尾实体**: Meridian
- **描述**: 药物的归经
- **示例**: 人参 → 归经 → 脾经
- **预期数量**: ~200-400
- **数据来源**: OWL 中"归...经"关系（13次使用）

#### 10. has_effect (功效)
- **头实体**: Herb / Prescription
- **尾实体**: Effect (文本描述或关联到 Therapy)
- **描述**: 药物或方剂的功效
- **示例**: 人参 → 功效 → 补气
- **预期数量**: ~300-500
- **数据来源**: OWL 中"作用"关系（107次使用）

### 养生相关关系

#### 11. uses_method (采用方法)
- **头实体**: Disease / Syndrome
- **尾实体**: WellnessMethod
- **描述**: 疾病或体质适用的养生方法
- **示例**: 气虚体质 → 采用方法 → 饮食养生
- **预期数量**: ~200-400

#### 12. has_operation (操作方法)
- **头实体**: WellnessMethod
- **尾实体**: String (文本)
- **描述**: 养生方法的具体操作
- **示例**: 艾灸 → 操作方法 → "每穴灸10-15分钟"
- **预期数量**: ~300-500
- **数据来源**: OWL 中"操作方法"关系（146次使用）

#### 13. has_frequency (使用频次)
- **头实体**: WellnessMethod
- **尾实体**: String (文本)
- **描述**: 养生方法的使用频率
- **示例**: 食疗 → 使用频次 → "每日一次"
- **预期数量**: ~200-300
- **数据来源**: OWL 中"使用频次"关系（52次使用）

#### 14. has_contraint (禁忌)
- **头实体**: WellnessMethod / Prescription / Herb
- **尾实体**: String (文本)
- **描述**: 使用禁忌或注意事项
- **示例**: 某药 → 禁忌 → "孕妇慎用"
- **预期数量**: ~100-200
- **数据来源**: OWL 中"禁忌"关系（33次使用）

### 层级关系

#### 15. subclass_of (子类/上位词)
- **头实体**: Any Entity
- **尾实体**: Any Entity (同类型或父类)
- **描述**: 实体的分类层级关系
- **示例**: 香砂六君子汤 → 子类 → 补气剂
- **预期数量**: ~600-800
- **数据来源**: OWL 中"上位词"和"下位词"关系（共657次使用）

### 时空关系

#### 16. related_to_season (相关节气/时节)
- **头实体**: WellnessMethod / Disease
- **尾实体**: String 或 Season 实体
- **描述**: 与季节、节气的关系
- **示例**: 春季养生 → 相关节气 → 春分
- **预期数量**: ~100-200
- **数据来源**: OWL 中"节气"关系（35次使用）

#### 17. related_to_time (时间相关)
- **头实体**: WellnessMethod
- **尾实体**: String (时辰描述)
- **描述**: 养生方法的适宜时间
- **示例**: 晨练 → 时间相关 → "早晨 5-7 点"
- **预期数量**: ~50-100

### 其他关系

#### 18. conceptually_related (概念相关)
- **头实体**: Any Entity
- **尾实体**: Any Entity
- **描述**: 概念上的关联（通用关系）
- **示例**: 气 → 概念相关 → 血
- **预期数量**: ~200-300
- **数据来源**: OWL 中"概念上相关"等关系

## 💡 设计说明

### 1. 为什么选择这些实体类型？

- **基于数据驱动**: 从 OWL 本体的 5,155 个实例中统计分析得出
- **覆盖全面**: 涵盖中医养生、临床诊疗的各个方面
- **实用性强**: 支持问答系统的常见查询场景

### 2. 为什么选择这些关系类型？

- **高频优先**: 选择 OWL 中使用频率最高的关系
- **语义清晰**: 每种关系都有明确的语义定义
- **易于理解**: 关系名称直观，便于后续开发

### 3. 与硕士论文的差异

| 项目 | 硕士论文要求 | 本设计 |
|------|------------|--------|
| 实体数量 | 12 类 | 10 类 |
| 关系数量 | 17 种三元组 | 18 种关系 |
| 侧重点 | 临床诊疗 | 养生 + 诊疗 |
| 灵活性 | 严格遵循 | 灵活调整 |

**说明**: 作为本科毕设，可以根据实际数据情况灵活调整，关键是自圆其说。

### 4. 可扩展性

- 后续可以添加更多实体类型（如：病因、病机等）
- 可以增加更细粒度的关系
- 可以融合 medical.json 的数据进行扩展
