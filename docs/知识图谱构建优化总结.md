## 🎯 二、核心问题分析

### 2.1 初始问题

#### 问题 1：实体分类错误
**现象**：大量书名被错误分类为 Disease、Herb、Meridian 等类型
```
错误示例：
- Disease: 《万病回春》《伤寒六经主症》（实际是书籍）
- Herb: 《本草纲目》《医学衷中参西录》（实际是书籍）
- Meridian: 《内经要旨》《奇经八脉考》（实际是书籍）
```

**原因**：关键词匹配规则过于简单，`《` 和 `》` 符号未被正确处理

#### 问题 2：关系数量极少
**现象**：仅 35 个关系，无法支撑复杂查询

**原因分析**：
1. OWL 本体中大部分关系指向"字面量"（文本属性），而非实体节点
2. "下位词/上位词"关系（共 657 个）的客体未被正确识别为实体
3. 大量有价值的概念被过滤掉

#### 问题 3：部分实体类型数据不足
**现象**：
- Herb（药物）仅 28 个
- Meridian（经络）仅 21 个
- WellnessMethod（养生方法）仅 30 个

**影响**：无法支撑完整的中医知识查询

## 🔧 三、优化策略与实施

### 3.1 策略一：修正实体分类规则

#### 实施方案
1. **优先识别书名**
   ```python
   def is_book_title(self, label: str) -> bool:
       return '《' in label or '》' in label
   
   def classify_entity(self, label: str) -> str:
       if self.is_book_title(label):
           return 'Classic'  # 所有书名归类为典籍
   ```

2. **引入 Concept 兜底机制**
   ```python
   # 对于无法精确分类但有意义的中文标签
   if len(label) >= 2 and any('\u4e00' <= c <= '\u9fff' for c in label):
       return 'Concept'  # 保留为通用概念
   ```

#### 效果
- ✅ 消除了书名误分类问题
- ✅ 保留了 2,584 个有价值的概念节点
- ✅ 为后续重新分类奠定基础

---

### 3.2 策略二：扩展关系映射

### 3.3 策略三：字面量转实体

#### 实施方案
将高频字面量转换为实体节点：

```python
literal_to_entity_rules = {
    '有症状': 'Symptom',      # 症状文本 → Symptom 节点
    '对应证候': 'Syndrome',   # 证候文本 → Syndrome 节点
    '功效': 'Effect',         # 功效文本 → Effect 节点
    '作用': 'Effect',         # 作用文本 → Effect 节点
    '来源': 'Classic',        # 书名 → Classic 节点
    '出自': 'Classic',        # 书名 → Classic 节点
}
```

**处理逻辑**：
1. 检测关系是否属于可转换类型
2. 提取字面量值（支持逗号、分号分隔的多值）
3. 创建对应的实体节点
4. 建立关系连接

#### 效果
- ✅ 新增 142 个实体节点
- ✅ FROM_CLASSIC 关系：154 个
- ✅ HAS_EFFECT 关系：250 个

### 4.2 典型查询示例

#### 示例 1：疾病的完整信息（新增能力）
```cypher
MATCH (d:Disease {name: '胃痛'})
```

## 💡 五、关键技术洞察

### 5.1 OWL 本体的特点

1. **关系稀疏性**
   - 5,155 个个体，仅约 1,000 个有效关系
   - 大部分关系指向字面量（属性），而非实体

2. **层级结构丰富**
   - "下位词/上位词"关系占主导（657 个）
   - 适合构建概念分类体系

3. **领域专有概念多**
   - 大量功法名称、理论术语
   - 需要灵活的分类策略

### 5.2 Concept 兜底策略的价值

**优势**：
- ✅ 避免有价值节点的丢失
- ✅ 保留本体完整性
- ✅ 为后续细化留有余地

**劣势**：
- ⚠️ 孤立节点较多（1,977 个）
- ⚠️ 关系密度偏低

**适用场景**：
- 本科毕设：完全足够
- 生产环境：需进一步优化

#### 实体分类
```python
def classify_entity(self, label: str) -> str:
    # 1. 优先识别书名
    if self.is_book_title(label):
        return 'Classic'
    
    # 2. 关键词匹配
    for entity_type, keywords in self.entity_categories.items():
        if any(kw in label for kw in keywords):
            return entity_type
    
    # 3. Concept 兜底
    if len(label) >= 2 and any('\u4e00' <= c <= '\u9fff' for c in label):
        return 'Concept'
    
    return 'Other'
```
