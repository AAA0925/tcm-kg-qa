### Step 3: 构建知识图谱

**运行构建脚本**:

```bash
cd backend

# 基本用法
python build_up_graph.py \
  --website http://localhost:7474 \
  --user neo4j \
  --password your_password \
  --dbname neo4j

# 完整参数
python build_up_graph.py \
  --website http://localhost:7474 \
  --user neo4j \
  --password your_password \
  --dbname neo4j \
  --data data/medical_new_2.json \
  --export-dir data
```

**参数说明**:
| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--website` | ✅ | - | Neo4j 地址 |
| `--user` | ❌ | `neo4j` | 用户名 |
| `--password` | ✅ | - | 密码 |
| `--dbname` | ❌ | `neo4j` | 数据库名 |
| `--data` | ❌ | `data/medical_new_2.json` | 数据文件路径 |
| `--export-dir` | ❌ | `data` | 导出目录 |

## 📊 构建过程

脚本会自动执行以下步骤:

### 1️⃣ 创建约束和索引
```cypher
CREATE CONSTRAINT disease_name FOR (d:Disease) REQUIRE d.name IS UNIQUE
CREATE CONSTRAINT symptom_name FOR (s:Symptom) REQUIRE s.name IS UNIQUE
...

CREATE INDEX disease_description FOR (d:Disease) ON (d.description)
CREATE INDEX herb_effects FOR (h:Herb) ON (h.effects)
...
```

### 2️⃣ 导入疾病实体
- 读取 `medical_new_2.json`
- 创建 Disease 节点，包含所有属性

### 3️⃣ 导入其他实体
自动识别并创建以下类型的实体:
- Drug (药品)
- Food (食物)
- Check (检查项目)
- Department (科室)
- Symptom (症状)
- Cure (治疗方法)
- Producer (在售药品)

### 4️⃣ 创建关系
根据数据集创建 11 类关系:
```cypher
(d:Disease)-[:HAS_SYMPTOM]->(s:Symptom)
(d:Disease)-[:COMMON_DRUG]->(drug:Drug)
(d:Disease)-[:RECOMMAND_DRUG]->(drug:Drug)
(d:Disease)-[:DO_EAT]->(food:Food)
(d:Disease)-[:NO_EAT]->(food:Food)
(d:Disease)-[:RECOMMAND_EAT]->(food:Food)
(d:Disease)-[:NEED_CHECK]->(check:Check)
(d:Disease)-[:CURE_WAY]->(cure:Cure)
(d:Disease)-[:ACOMPANY_WITH]->(d2:Disease)
(d:Disease)-[:BELONGS_TO]->(dept:Department)
(drug:Drug)-[:DRUGS_OF]->(producer:Producer)
```

### 5️⃣ 导出数据
自动在 `data` 目录下创建:
- `ent_aug/` - 实体文件目录
  - `disease.txt`
  - `drug.txt`
  - `symptom.txt`
  - ...
- `rel_aug.txt` - 关系文件

## ⚠️ 常见问题

### Q1: 连接 Neo4j 失败
**错误**: `Failed to establish connection`

**解决方案**:
1. 检查 Neo4j 是否运行：`docker ps` 或访问 http://localhost:7474
2. 确认用户名密码正确
3. 检查防火墙设置

### Q2: 内存不足
**错误**: `OutOfMemoryError`

**解决方案**:
1. 增加 Neo4j 堆内存配置
2. 分批导入数据（修改脚本中的 batch_size）

### Q3: 约束冲突
**错误**: `ConstraintViolationException`

**解决方案**:
- 这是正常的，说明数据已存在
- 脚本使用了 MERGE 会自动处理

### Q4: 导入速度慢
**优化建议**:
1. 使用 SSD 硬盘
2. 增加 Neo4j 缓存配置
3. 禁用日志（仅开发环境）:
   ```properties
   dbms.logs.query.enabled=false
   ```
